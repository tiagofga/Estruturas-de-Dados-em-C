# Algorithms and Big-O Notation

[Português](../Algoritmos.md) | **English**

## Overview

An algorithm is a finite sequence of well-defined steps used to solve a problem. In this repository, algorithms are studied together with their data structures and computational cost.

---

## Big-O notation

Big-O notation describes the upper bound of an algorithm's growth rate as the input size increases.

Examples:

```text
O(1)       constant
O(log n)   logarithmic
O(n)       linear
O(n log n) linearithmic
O(n²)      quadratic
O(2ⁿ)      exponential
```

---

## Practical interpretation

For small inputs, implementation details may dominate. For large inputs, asymptotic complexity usually becomes the most important factor.
